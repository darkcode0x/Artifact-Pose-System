IOT EMBEDDED CONTROL ARCHITECTURE
No cloud mode: Mobile <-> Server <-> MQTT Broker <-> Raspberry Pi Edge Node (clean routed view)
Protocols: Mobile <-> Server (HTTPS or WebSocket), Server <-> Broker (MQTT), Broker <-> Raspberry Pi (MQTT), Raspberry Pi -> Server (HTTPS upload).
Execution path: cmd/device_id -> parse -> dedup by task_id TTL -> hardware/capture -> ack/status.
Design goal: deterministic embedded control, bounded memory idempotency, and clear bidirectional communication.
